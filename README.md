# 校园无人机起降点选址分析

本项目为仓库作者在《低空工程与技术概论》课程学习过程中完成的个人报告，旨在对校园内无人机起飞与降落候选场地的适宜性进行分析。项目通过读取 CSV 数据、计算加权总分，并结合候选点之间的间距修正生成最终双点组合推荐，从而对不同候选点进行系统比较。

## 项目概述

### 项目背景

在校园场景中，无人机不仅可以服务宿舍区，也可以服务教学区、综合服务区和物流相关区域。为了保证运行安全和服务效率，需要从多个候选场地中选择更适合作为起降点的位置。

### 项目目标

本项目的目标是：

- 读取候选场地评分数据
- 根据多个评价指标计算综合得分
- 对候选场地进行排序
- 通过图表展示不同场地的比较结果

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 运行主程序

在项目根目录下执行：

```bash
python src/main.py
```

程序会完成以下操作：

- 读取 `data/candidate_scores.csv`
- 校验数据合法性
- 计算每个候选点的加权总分
- 读取 `data/candidate_pair_distances.csv` 并计算双点组合的间距修正
- 在终端输出单点排序和双点组合排序
- 生成热力图、柱状图和雷达图

生成的图片将保存在 `figures/` 目录中。

## 评价方法

### 评价指标

项目使用以下 7 个评价指标：

- `service_distance`：与服务区的距离，越合适分数越高
- `logistics_distance`：与物流点的距离，越合适分数越高
- `openness`：场地开阔程度，越开阔分数越高
- `obstacle_risk`：障碍物风险，风险越低分数越高
- `crowd_risk`：人流干扰风险，风险越低分数越高
- `route_access`：航线通达性，越方便分数越高
- `operation_convenience`：运维便利性，越方便分数越高

### 评分标准

每个指标使用 `1` 到 `5` 的整数评分：

- `1`：很差 / 很不适合
- `2`：较差
- `3`：一般
- `4`：较好
- `5`：很好 / 很适合

分数越高，表示该候选点在该指标上的表现越好。

### 加权评分模型

项目使用加权总分方法对候选场地进行综合评价，权重如下：

- `service_distance`：`0.15`
- `logistics_distance`：`0.15`
- `openness`：`0.20`
- `obstacle_risk`：`0.15`
- `crowd_risk`：`0.15`
- `route_access`：`0.10`
- `operation_convenience`：`0.10`

综合得分计算方式为：

```text
total_score = 各指标分数 × 对应权重 后求和
```

其中，`openness` 权重最高，因为场地开阔程度通常对无人机安全起降较为重要。

### 双点间距修正

当项目需要从候选点中最终选出两个起降点时，程序会额外读取 `data/candidate_pair_distances.csv` 中的两两距离，并根据间距给组合分做修正：

- 距离过近：降低组合得分，避免两个点服务范围高度重叠
- 距离适中：不额外加减分
- 距离较远：适度加分，鼓励覆盖面更开

当前默认规则如下：

| 两点间距 | 间距评分 | 组合修正 |
| --- | ---: | ---: |
| ≤150 m | 1 | -0.70 |
| 150–300 m | 2 | -0.35 |
| 300–500 m | 3 | 0.00 |
| 500–700 m | 4 | +0.35 |
| >700 m | 5 | +0.70 |

最终双点组合分计算方式为：

```text
pair_score = 点位 A 单点评分 + 点位 B 单点评分 + 间距修正
```

## 可视化结果

项目会生成 3 张图表，并保存在 `figures/` 目录中。

### 1. 评分热力图

热力图展示所有候选点在 7 个评价指标上的原始分数。颜色越深表示得分越高，每个单元格中也会显示具体数值，便于直接比较。

![评分热力图](figures/heatmap.png)

### 2. 综合评分柱状图

柱状图展示每个候选点的加权总分，并按照总分从高到低排序。柱子上方会标出具体分值，方便查看最终排名。

![综合评分柱状图](figures/bar_chart.png)

### 3. 推荐双点组合雷达图

雷达图展示最终推荐双点组合的两个候选点，对比它们在 7 个评价指标上的表现，并在图中标注两点间距、间距修正和组合总分，帮助解释为什么该组合比“单点评分前二”更适合同时部署。

![推荐双点组合雷达图](figures/radar_chart.png)

如需查看更完整的结果说明和最终排名表，可参考 [results.md](docs/results.md)。

## 项目局限性

本项目适合作为校园无人机起降点选址的初步分析工具，但仍存在一些局限性：

- 候选点的评分依赖人工判断，因此带有一定作者本人的主观性。即使有统一评分细则，不同评价者对同一场地的理解和打分仍可能存在差异。
- 当前评分主要基于定性分析和离散分值，尚未接入实测距离、实时人流、风环境或 GIS 空间数据，因此结果更适合用于初步比较，而不是精细化决策。
- 加权模型中的权重是预先设定的，不同应用目标下权重可能需要调整。例如，更强调安全时，障碍物风险和人流风险的权重可能应进一步提高。
- 项目默认所有候选点都满足基本可选条件，而现实场景中还可能存在禁飞区、临时施工、管理权限限制等额外约束，这些因素目前未被完整纳入模型。

因此，本项目结果更适合作为方案筛选和课程演示的参考依据。若要用于更真实的部署决策，仍需要结合现场踏勘、更多客观数据和更细致的规则分析。
